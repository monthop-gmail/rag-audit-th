## A. ธรรมาภิบาลและขอบเขต

### TH-LRAG-GOV-001 — กรณีการใช้งานทางกฎหมายที่กำหนดไว้

| รายการ | รายละเอียด |
|--------|-----------|
| ระดับความรุนแรง | HIGH |
| สถานะ | **PARTIAL** |
| หลักฐาน | README, เทมเพลตกฎหมาย 4 รายการใน `legalTemplates.ts` |
| ข้อค้นพบ | กรณีการใช้งานที่อนุญาตถูกกำหนดโดยปริยายผ่านเทมเพลต (NDA, PDPA QA, ตรวจสอบ compliance, สรุปสัญญา) แต่กรณีการใช้งานที่ห้ามไม่ได้ระบุไว้ที่ใดเลย |
| การแก้ไข | จัดทำ `USAGE_POLICY.md` กำหนดกรณีการใช้งานที่อนุญาตและที่ห้าม |

### TH-LRAG-GOV-002 — ความรับผิดชอบของบุคคล

| รายการ | รายละเอียด |
|--------|-----------|
| ระดับความรุนแรง | HIGH |

| การแก้ไข | กำหนดและจัดทำเอกสารบทบาทเจ้าของทั้ง 4 ตำแหน่ง |

## B. ความถูกต้องตามกฎหมายของแหล่งข้อมูลและอำนาจทางกฎหมาย

### TH-LRAG-SRC-001 — ทะเบียนแหล่งข้อมูลกฎหมายที่เป็นทางการ

| รายการ | รายละเอียด |
|--------|-----------|
| ระดับความรุนแรง | CRITICAL |
| สถานะ | **FAIL** |
| หลักฐาน | ไฟล์ markdown 7 ไฟล์ใน `/knowledge-base/` จัดตามหมวดหมู่ ไม่มีทะเบียนแหล่งข้อมูลหรือ allowlist อย่างเป็นทางการ |
| ข้อค้นพบ | ไม่มีทะเบียนแหล่งข้อมูลที่ได้รับอนุมัติ มีไฟล์อยู่แต่ไม่มีการติดตามที่มา |
| การแก้ไข | จัดทำทะเบียนแหล่งข้อมูลระบุที่มา อำนาจ และสถานะการอนุมัติของแต่ละเอกสาร |

### TH-LRAG-SRC-002 — การให้ความสำคัญกับแหล่งข้อมูลทางการ

| รายการ | รายละเอียด |
|--------|-----------|
| ระดับความรุนแรง | CRITICAL |
| สถานะ | **FAIL** |
| หลักฐาน | `retriever.ts` ส่งคืน top-K ตาม cosine similarity เท่านั้น ไม่มีการถ่วงน้ำหนักลำดับความสำคัญของแหล่งข้อมูล |
| ข้อค้นพบ | ไม่มีกลไกให้ความสำคัญกับแหล่งข้อมูลทางการของรัฐบาลไทยมากกว่าเนื้อหาไม่เป็นทางการ |
| การแก้ไข | เพิ่มการถ่วงน้ำหนักลำดับความสำคัญของแหล่งข้อมูลในตรรกะการจัดอันดับ |

### TH-LRAG-SRC-003 — metadata ที่มาของแหล่งข้อมูล

| รายการ | รายละเอียด |
|--------|-----------|
| ระดับความรุนแรง | CRITICAL |
| สถานะ | **FAIL** |
| หลักฐาน | metadata ของ chunk: `{ source, title, heading, category, position }` ขาด: URL แหล่งข้อมูล, วันที่เผยแพร่, วันที่มีผลบังคับใช้, วันที่นำเข้า |
| ข้อค้นพบ | ติดตามเฉพาะชื่อไฟล์และหมวดหมู่ ขาดฟิลด์ที่มาที่สำคัญ |
| การแก้ไข | ขยาย metadata schema ด้วย `source_url`, `publication_date`, `effective_date`, `ingestion_date` |

### TH-LRAG-SRC-004 — metadata สถานะของกฎหมาย

| รายการ | รายละเอียด |
|--------|-----------|
| ระดับความรุนแรง | CRITICAL |
| สถานะ | **FAIL** |
| หลักฐาน | ไม่มีฟิลด์ `status` (ใช้บังคับ/แก้ไข/ยกเลิก/ร่าง) ใน metadata schema ของ chunk |
| ข้อค้นพบ | ไม่มีการติดตามสถานะกฎหมาย ระบบไม่สามารถแยกแยะกฎหมายที่ถูกยกเลิกจากกฎหมายที่ใช้บังคับ |
| การแก้ไข | เพิ่มฟิลด์ `law_status` ใน metadata schema และเติมข้อมูล |

### TH-LRAG-SRC-005 — การติดตามสายการแก้ไข

| รายการ | รายละเอียด |
|--------|-----------|
| ระดับความรุนแรง | HIGH |
| สถานะ | **FAIL** |
| หลักฐาน | ไม่มีการเชื่อมโยงเวอร์ชันระหว่างเอกสาร ไม่มีการติดตามการแก้ไขใน metadata หรือตรรกะการนำเข้า |
| ข้อค้นพบ | กฎหมายที่ถูกแก้ไขไม่สามารถสืบย้อนไปยังเวอร์ชันก่อนหน้า |
| การแก้ไข | เพิ่ม metadata สายการแก้ไขเชื่อมโยงบทบัญญัติที่แก้ไขกับเวอร์ชันก่อนหน้า |

### TH-LRAG-SRC-006 — การตรวจสอบสิทธิ์การใช้งานแหล่งข้อมูล

| รายการ | รายละเอียด |
|--------|-----------|
| ระดับความรุนแรง | HIGH |

| หลักฐาน | ฐานความรู้ปัจจุบันมีเฉพาะเอกสาร markdown ที่จัดทำภายในและเทมเพลต |
| ข้อค้นพบ | ไม่พบเนื้อหาที่มีลิขสิทธิ์ของบุคคลภายนอก |

### TH-LRAG-SRC-007 — การแบ่งหมวดหมู่แหล่งข้อมูล

| รายการ | รายละเอียด |
|--------|-----------|
| ระดับความรุนแรง | HIGH |
| สถานะ | **PARTIAL** |
| หลักฐาน | ฟิลด์ `category` มีค่า: `laws`, `contracts`, `policies`, `faq` ไม่มีการแยกแยะระหว่างกฎหมายหลัก กฎหมายลำดับรอง คำพิพากษา บทวิเคราะห์ หรือเนื้อหาที่ AI สร้าง |
| ข้อค้นพบ | มีการแบ่งหมวดหมู่เบื้องต้นแต่ taxonomy หยาบเกินไปสำหรับข้อกำหนดทางกฎหมาย |
| การแก้ไข | ขยาย taxonomy ให้รวมประเภทแหล่งข้อมูลทางกฎหมาย (กฎหมายหลัก, กฎระเบียบ, คำพิพากษา, บทวิเคราะห์, นโยบายภายใน) |

|--------|-----------|

| หลักฐาน | ไม่มีการระบุป้ายร่าง/ประกาศใช้ใน metadata ไม่มีกฎการสืบค้นเพื่อป้องกันการอ้างอิงร่างกฎหมายเป็นกฎหมายที่มีผลบังคับใช้ |

## C. คุณภาพเนื้อหากฎหมายไทย

### TH-LRAG-LAW-001 — ความละเอียดของการอ้างอิง

| รายการ | รายละเอียด |
|--------|-----------|
| ระดับความรุนแรง | CRITICAL |
| สถานะ | **PARTIAL** |
| หลักฐาน | `chunker.ts` แบ่งตามหัวข้อ markdown (##/###) ซึ่งตรงกับมาตรา ฟิลด์ `heading` เก็บข้อมูลระดับมาตรา แต่การอ้างอิงในผลลัพธ์แสดงเฉพาะ `title (source)` ไม่ระบุมาตราเฉพาะ |
| ข้อค้นพบ | มี metadata ระดับหัวข้อแต่ไม่ได้แสดงเป็นการอ้างอิงแบบละเอียดในคำตอบ |
| การแก้ไข | รวม `heading` (มาตรา/ข้อ) ในรูปแบบการอ้างอิงผลลัพธ์ |

### TH-LRAG-LAW-002 — การรับรู้วันที่มีผลบังคับใช้

|--------|-----------|
| ระดับความรุนแรง | CRITICAL |
| สถานะ | **FAIL** |

| ข้อค้นพบ | ระบบไม่มีแนวคิดเรื่องวันที่มีผลบังคับใช้ |
| การแก้ไข | เพิ่ม `effective_date` ใน metadata; นำการสืบค้นที่คำนึงถึงวันที่มาใช้ |

### TH-LRAG-LAW-003 — การรับรู้การยกเลิกและแก้ไข

| การแก้ไข | เพิ่มการติดตามสถานะกฎหมายและกรองเนื้อหาที่ถูกยกเลิกออกจากผลลัพธ์เริ่มต้น |
